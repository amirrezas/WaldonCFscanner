# Contributing to WaldonCFscanner

First off, thank you for considering contributing to WaldonCFscanner! It's people like you that make open-source tools great. 

## üêõ Bug Reports
If you find a bug or experience a crash, please open an issue on GitHub. To help us fix it quickly, please include:
1. Your Operating System (Windows, Android, Linux, macOS).
2. The version of WaldonCFscanner you are using.
3. The **`scanner_error.log`** file (you can generate this by clicking the "Save Log" button in the app). 
4. A clear description of what went wrong.

## ‚ú® Feature Requests
Have an idea to make the scanner faster, more accurate, or easier to use? We would love to hear it! Open an issue and use the `enhancement` tag to explain your idea.

## üíª Pull Requests (Code Contributions)
We welcome Pull Requests! If you want to fix a bug or add a feature, please follow these steps:
1. **Fork** the repository.
2. Create a new branch for your feature (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a **Pull Request**.

Please ensure your code keeps the asynchronous pipeline efficient and doesn't break the existing 4-stage Xray verification logic.
